Module: platform/core/base/pin

abstract description of a pin object. pins are for transference of data or information between nodes, for example when a node wants to notify another node of something that happened, pass some data, etc. all data and information getting in and out of nodes are transferred using pins, including all inputs, outputs, etc.

(inner, constant) PinEvents :string

the events that can happen to all pins. specific pin types have their own specific types.
Type:
  • string
Properties:
Name Type Description
connect string denotes the event of a pin being connected to another pin.
disconnect string denotes the event of a pin being disconnected from another pin.
activate string denotes the event of a pin being activated, i.e. is being used for communication between two nodes.
reset string denotes the event of a node being reset to its base state. connections of a pin will remain in-tact and only its activation state will be reset. specific pin types might also do further clean up in the event of a reset.
